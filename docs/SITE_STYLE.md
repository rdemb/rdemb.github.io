# Site Style

This is the working style guide for future edits to the public site.

## Writing Skill

- Start with the thing being done, not with a persona.
- Prefer verbs over identity labels.
- Keep claims narrow and testable.
- Say what failed when the failure teaches something.
- Use first person when it makes the sentence more honest.
- Avoid corporate filler, launch language, and generic inspiration.
- One paragraph should usually carry one thought.
- Caveats stay near the result, not hidden at the bottom.

## UI Skill

- The site is a notebook, not a landing page.
- Navigation should expose the main paths without hiding them behind a menu.
- Cards are for repeated objects: projects, posts, and section choices.
- Use one quiet surface, one dark text color, and small accents.
- Mobile touch targets must stay easy to tap.
- Decorative effects must earn their weight in readability or orientation.
- Add structure before adding ornament.

## Homepage Rule

MOCPS is one project on the site, not the identity of the whole site.
The homepage should make room for future projects and reflections.
